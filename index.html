REF · ENG For technical reviewers

What I personally designed, built, and defended.

Auditly is a solo build. These are the load-bearing engineering decisions — the parts I'd want to walk you through in an interview — not a feature list.

Multi-tenancy

DB-enforced tenant isolation

Postgres row-level security on every tenant table, scoped through a single query helper. A cross-tenant read returns 404, not 403 — isolation is a database invariant, not application discipline that can be forgotten in one endpoint.

Async pipeline

Document processing with failure recovery

Uploads run through a Redis/BullMQ worker chain (scan → parse → classify → extract → bridge). Every stage is idempotent; an unrecoverable error lands the document in a terminal failed state with a reason instead of silently hanging or double-counting on retry.

Trust gate

Reconciliation before acceptance

Extracted bank figures are only trusted when the statement's own roll-forward ties out to the penny (beginning + deposits − withdrawals = ending). If it doesn't close, the value is flagged for review — the deterministic invariant is the gate, upstream of any model.

Grounded AI

LLM as a validated, non-authoritative layer

The copilot only ever sees an allow-listed grounding bundle and defers to the deterministic engine; LLM-extracted values are re-validated (tie-out + provenance) before they count. Backed by a no-leak / no-fabrication safety-eval suite.

Deploy / ops

Containerized, deployed, self-healing demo

The full stack (API · workers · Postgres · Redis · object storage · authorization · document OCR) runs from one Compose file behind Caddy auto-TLS on a VPS, with per-IP rate limiting, a hard LLM spend cap, and an unattended nightly reset that restores a clean, seeded demo.

Observability

Cost + audit instrumentation

Every model call is logged with provider, model, tokens, latency, and USD cost; every capture emits to an append-only audit trail tagged with method, model, and confidence. You can answer "how was this number produced, and what did it cost" for any figure.

Honest scope

Production-style, not production-scale. This is a portfolio system: real architecture, real reconciliation, real deployment — running on a single small VPS with synthetic demo data. It is not carrying live customer traffic or regulated financial data. Where something is simulated (e.g. the free local-LLM fallback, the seeded engagements), the page says so.

REF · A The problem

Audit fieldwork starts with weeks of chasing paper.

The painful part of an audit isn't judgment — it's collecting, classifying, and tying out the client's evidence before any real work can begin. Auditly moves that work to the front, and automates the parts a machine should own.

Today

Manual, back-and-forth, error-prone

PBC request lists over email. Statements re-keyed by hand. Tie-outs done in spreadsheets. The client shows up disorganized; the auditor spends the first weeks on logistics, not assurance.

With Auditly

Organized and half–tied-out on arrival

The client self-serves onboarding, evidence is extracted and reconciled on upload, and a readiness check tells them what's missing before fieldwork. The auditor opens a file that's already substantiated.

REF · B What it does

Four things worth building carefully.

Extraction is table stakes by now. The defensible parts are how the numbers get verified, how the AI stays honest, and how every artifact becomes durable evidence.

B-1

Deterministic-first extraction

Bank-statement specialists pull structured fields and reconcile them (tie-outs) before trusting them — most documents need no LLM at all. When one does, the call is metered and hard-capped, with a free local model as automatic fallback so the demo never breaks or overspends.

$0 on the deterministic path
B-2

Grounded copilot, allow-listed context

One reusable spine powers copilots on four surfaces. The model only ever sees an allow-listed grounding bundle and defers to the deterministic engine — it surfaces figures the engine validated, not ones it made up, and can't reach the auditor's internals. A safety-eval suite checks for leakage and fabrication.

no-leak / no-fabrication evals
B-3

Evidence substrate + signed audit trail

Every capture — extraction, verification, correction, metric — emits to an append-only log tagged with method, model, and consent. Control-test records are Ed25519-signed per tenant, so a modified or forged historical record fails signature verification — genuine tamper-evidence, not just an append-only claim.

append-only · Ed25519-signed records
B-4

Two-sided and multi-tenant

The same platform serves the audit firm and the client from distinct, role-aware workspaces. Isolation is enforced at the database with Postgres row-level security — a cross-tenant read returns 404, not 403.

RLS on every tenant table

REF · D The extraction engine

A real statement in — reconciled transactions out, with its work shown.

The engine is deterministic-first: layout parsing and bank-specific parsers do the work, and every figure is reconciled (tied out) before it counts. The LLM is break-glass only — it fills the few fields the deterministic layers abstain on, never invents a number, and the result says exactly which fields it touched.

1
Read the document docling
Layout-aware parse of the PDF into structured Markdown — tables, columns, and the statement's own transaction register all survive.
2
Tier 1 capture deterministic
Anchor/regex extractors pull institution, statement period, account #, and ending balance — each with a calibrated confidence band.
3
Bank specialist deterministic
A per-bank profile (e.g. Capital One 360) resolves the identity fields from that bank's exact layout — and skips the LLM entirely when it's confident.
4
Register parser deterministic
Reads the transaction register line-by-line and derives the deposit / withdrawal totals — accepted only when the register's own opening & closing rows reconcile.
Tie-out validation the trust gate
beginning + deposits − withdrawals = ending, to the penny. If it doesn't close, the figure is flagged for review — never silently trusted.
↯
LLM break-glass only if needed
For the few fields the deterministic layers abstain on, a hard-capped Haiku call (with a free local model as automatic fallback) fills the gap — and the reveal names those fields explicitly.
→
Transactions + evidence
Reconciled rows become testable transactions, and every capture emits to the append-only audit trail tagged with method, model, and confidence.
